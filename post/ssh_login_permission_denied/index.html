ssh 免密钥登录提示 permission denied 的问题

ssh 密钥登录提示Permission denied (publickey,gssapi-keyex,gssapi-with-mic) 的解决办法（之一）：

远程服务器更换镜像之后，ssh 使用配置项登录提示如下：

Permission denied (publickey,gssapi-keyex,gssapi-with-mic)

一番搜索之后，原因是 ssh 尝试登录非 root 用户，比如 A 用户： ssh -i <path_to_your_prikey> A@your_remote_ip_addr 。提示：Permission denied (publickey,gssapi-keyex,gssapi-with-mic)。有可能是在远程服务器上，A用户目录下：/home/A 没有相关的 .ssh 目录和 .ssh/authorized_keys 公钥文件。按常规的服务器配置密钥登录步骤，在 A 用户家目录下配置好密钥即可。
